Big O, Big Θ and Big Ω

Introduction

This page is about the mathematical way of obtaining a function's Big O. For the analytical one, see Measuring Algorithm Growth (Level 2).

In the previous lesson, we introduced to you the concept of function growth. In this lesson, we will learn about Big OOO, Big Θ\ThetaΘ, and Big Ω\OmegaΩ, and what each of them mean.

In CS, Big OOO, Big Θ\ThetaΘ, and Big Ω\OmegaΩ all represents the worst, average and best time complexity of an algorithm, respectively. Here, we will have two functions: f(x)f(x)f(x), our function to estimate, and g(x)g(x)g(x), the "bound" on the growth rate of f(x)f(x)f(x). Each "Big X" has it's own specific bounds and ways of how f(x)f(x)f(x) should behave relative to g(x)g(x)g(x) as x→∞x \to \inftyx→∞.

Return to Limits

See what I did there? "as x→∞x \to \inftyx→∞". To better understand how f(x)f(x)f(x) behaves relative to g(x)g(x)g(x), we will be returning to the concept of limits.

In mathematics, a limit is the value or behavior of a function as it's argument (xxx) approaches a certain value, and in this lesson, we are trying to observe how f(x)f(x)f(x) behaves relative to g(x)g(x)g(x) as x→∞x \to \inftyx→∞.

Individually, the limits of both these functions are:

lim⁡x→+∞f(x)=+∞ and lim⁡x→+∞g(x)=+∞\lim_{x\to+\infty} f(x) = +\infty \text{ and } \lim_{x\to+\infty} g(x) = +\infty x→+∞lim​f(x)=+∞ and x→+∞lim​g(x)=+∞

However, by dividing them against each other, we can algebraically redefine the resulting function in such a way that allows us to see the behavior of f(x)f(x)f(x) against g(x)g(x)g(x).

To put it simply, we are trying to evaluate the growth of f(x)f(x)f(x) relative to g(x)g(x)g(x) as x→∞x \to \inftyx→∞. We can think about this concept with a simple limit, defined as:

lim⁡x→+∞f(x)g(x)\lim_{x\to+\infty} \frac{f(x)}{g(x)} x→+∞lim​g(x)f(x)​

Here, we compare the growth of f(x)f(x)f(x) to g(x)g(x)g(x) as they approach ∞\infty∞. This works because, as x→∞x \to \inftyx→∞, the fastest growing term dominates over the others, making the lower terms obsolete as they approach ∞\infty∞.

To explain this further, we need a limit theorem too, namely:

lim⁡x→+∞1xr=0for r≥1\lim_{x\to+\infty} \frac{1}{x^r} = 0 \hspace{1em} \text{for} ~r \geq 1 x→+∞lim​xr1​=0for r≥1

Now that we have that aside, all we have to do is, well, evaluate the limit.

Let's say our functions are f(x)=x2+4x+16f(x) = x^2+4x+16f(x)=x2+4x+16 and g(x)=x2g(x) = x^2g(x)=x2. Now, when we substitute them in our limit:

lim⁡x→+∞f(x)g(x)\lim_{x\to+\infty} \frac{f(x)}{g(x)} x→+∞lim​g(x)f(x)​

lim⁡x→+∞x2+4x+16x2\lim_{x\to+\infty} \frac{x^2+4x+16}{x^2} x→+∞lim​x2x2+4x+16​

Can we evaluate this limit yet? No, because if we substitute xxx as +∞+\infty+∞, we'd get an indeterminate form of ∞∞\frac{\infty}{\infty}∞∞​, so we need to redefine the function algebraically.

lim⁡x→+∞x2(1+4x+16x2)x2(1)\lim_{x\to+\infty} \frac{x^2(1+\frac{4}{x}+\frac{16}{x^2})}{x^2(1)} x→+∞lim​x2(1)x2(1+x4​+x216​)​

This is a key point in understanding how Big O works. When we redefine this function, we divide the numerator and denominator by the "largest degree" of xxx in the denominator. In this way, the denominator evaluates to 111, leaving the value to the "largest degree" of xxx in the numerator.

lim⁡x→+∞1+4x+16x21\lim_{x\to+\infty} \frac{1+\frac{4}{x}+\frac{16}{x^2}}{1} x→+∞lim​11+x4​+x216​​

lim⁡x→+∞1+4x+16x2\lim_{x\to+\infty} 1+\frac{4}{x}+\frac{16}{x^2} x→+∞lim​1+x4​+x216​

Now, we can substitute xxx as +∞+\infty+∞.

1+4+∞+16(+∞)21+\frac{4}{+\infty}+\frac{16}{(+\infty)^2} 1++∞4​+(+∞)216​

We use the limit theorem from before.

1+0+01+0+0 1+0+0

11 1

Therefore, we can say that.

lim⁡x→+∞x2+4x+16x2=1\lim_{x\to+\infty} \frac{x^2+4x+16}{x^2} = 1 x→+∞lim​x2x2+4x+16​=1

Since it's 111, the growth of f(x)f(x)f(x) relative to g(x)g(x)g(x) as x→∞x \to \inftyx→∞ is like g(x)g(x)g(x).

Big O notation

Okay, back to the topic at hand. Big O represents the worst-case time complexity of an algorithm, where f(x)f(x)f(x) is our algorithm, and g(x)g(x)g(x) is our "upper bound" (this will be important later!). From our example above, it's quite evident why we don't include other "lower" constants in Big O notation.

We will return to our previous example. It is evident that as x→∞x \to \inftyx→∞, the values for 4x4x4x and 161616 become negligible.

Formally, to show that f(x)∈O(g(x))f(x) \in O(g(x))f(x)∈O(g(x)), we prove that:

f(x)≤c⋅g(x)for all x≥x0,where c>0 and x0 is a constantf(x) \leq c \cdot g(x) \hspace{1em} \text{for all}~x \geq x_0, \text{where } c > 0~\text{and}~x_0~\text{is a constant} f(x)≤c⋅g(x)for all x≥x0​,where c>0 and x0​ is a constant

Remember when we we're tasked to change nnn and ccc? Well, this is it, except the way you change those values can be errorneous. When you change nnn (in this case, xxx), you don't change ccc with it. Instead, you provide values for all x≥x0x \geq x_0x≥x0​ with a specified constant ccc, then increase ccc after. Our definition also states "for all", meaning that a single false indicates that f(x)∉O(g(x))f(x) \notin O(g(x))f(x)∈/O(g(x)).

For example, let's try proving that O(n2)∈O(n)O(n^2) \in O(n)O(n2)∈O(n):

If f(x)∈O(g(x)), then f(x)≤c⋅g(x) for all x≥x0,where c>0 and x0 is a constant\text{If } f(x) \in O(g(x)) \text{, then } f(x) \leq c \cdot g(x) \text{ for all}~x \geq x_0, \text{where } c > 0~\text{and}~x_0~\text{is a constant} If f(x)∈O(g(x)), then f(x)≤c⋅g(x) for all x≥x0​,where c>0 and x0​ is a constant

f(x)≤c⋅g(x)f(x) \leq c \cdot g(x) f(x)≤c⋅g(x)

n2≤c⋅nn^2 \leq c \cdot n n2≤c⋅n

n2n≤c⋅nn\frac{n^2}{n} \leq \frac{c \cdot n}{n} nn2​≤nc⋅n​

n≤cn \leq c n≤c

There exists no constant c such that n≤c. Therefore, O(n2)∉O(n)\text{There exists no constant } c \text{ such that } n \leq c \text{. Therefore, } O(n^2) \notin O(n) There exists no constant c such that n≤c. Therefore, O(n2)∈/O(n)

Why is it false (∉\notin∈/)? This is because O(n)O(n)O(n) can't be an upper bound for O(n2)O(n^2)O(n2), and this is right, since a linear function obviously grows lesser than a quadratic one.

An "upper bound" is the function describing the worst-case growth, in this case: O(n)O(n)O(n). However, it's impossible for O(n)O(n)O(n) to be an upper bound since the growth of the function is already O(n2)O(n^2)O(n2), which is already greater than that upper bound.

For instance, if our upper bound is a function like g(x)=n3g(x) = n^3g(x)=n3, h(x)=3nh(x) = 3^nh(x)=3n, or j(x)=(n!)j(x) = (n!)j(x)=(n!), then it would be valid as the growth of those functions are greater than that of f(x)=n2f(x) = n^2f(x)=n2.

From this, we know that g(x)g(x)g(x) (our upper bound) should obviously be greater than or equal to f(x)f(x)f(x) for it to be a possible upper bound for f(x)f(x)f(x).

Noting this and our limit definition, we can say that:

If lim⁡x→+∞f(x)g(x)≤1, then the function O(g(x)) can be considered an upper bound for f(x)\text{If } \lim_{x\to+\infty} \frac{f(x)}{g(x)} \leq 1 \text{, then the function } O(g(x)) \text{ can be considered an upper bound for } f(x) If x→+∞lim​g(x)f(x)​≤1, then the function O(g(x)) can be considered an upper bound for f(x)

This limit also fits our requirements, as if it's equal to 111, it would indicate that f(x)f(x)f(x) and g(x)g(x)g(x) grow at the same rate as x→∞x \to \inftyx→∞. If it's less than 111, or 000 since xxx would be approaching ∞\infty∞, we know that the denominator: g(x)g(x)g(x) grows faster than the numerator: f(x)f(x)f(x), thus, allowing it to be considered as an upper bound.

I suggest however that you'd use the proof instead (since it might be asked for). The limit definition is just a simpler way to think about this.

Big Θ notation

Big Θ\ThetaΘ, or Big Theta, represents the average-case time complexity of an algorithm, where f(x)f(x)f(x) is our algorithm, and g(x)g(x)g(x) is our "tight bound".

Formally, to show that f(x)∈Θ(g(x))f(x) \in \Theta(g(x))f(x)∈Θ(g(x)), we prove that:

c1⋅g(x)≤f(x)≤c2⋅g(x)for all x≥x0,where c1,c2>0 and x0 is a constantc1 \cdot g(x) \leq f(x) \leq c2 \cdot g(x) \hspace{1em} \text{for all}~x \geq x_0, \text{where } c1, c2 > 0~\text{and}~x_0~\text{is a constant} c1⋅g(x)≤f(x)≤c2⋅g(x)for all x≥x0​,where c1,c2>0 and x0​ is a constant

Now, why the introduction of c1c1c1 and c2c2c2? This is because Big Θ\ThetaΘ is a tight bound on the growth rate of the function. To put it simply, f(x)f(x)f(x) must match the growth rate of g(x)g(x)g(x) while ignoring the constants.

For example, let's try proving that Θ(3n2)∈Θ(n2)\Theta(3n^2) \in \Theta(n^2)Θ(3n2)∈Θ(n2):

If f(x)∈Θ(g(x)), then c1⋅g(x)≤f(x)≤c2⋅g(x) for all ... \text{If } f(x) \in \Theta(g(x)) \text{, then } c1 \cdot g(x) \leq f(x) \leq c2 \cdot g(x) \text{ for all ... } If f(x)∈Θ(g(x)), then c1⋅g(x)≤f(x)≤c2⋅g(x) for all ... 

c1⋅g(x)≤f(x)≤c2⋅g(x)c1 \cdot g(x) \leq f(x) \leq c2 \cdot g(x) c1⋅g(x)≤f(x)≤c2⋅g(x)

c1⋅n2≤3n2≤c2⋅n2c1 \cdot n^2 \leq 3n^2 \leq c2 \cdot n^2 c1⋅n2≤3n2≤c2⋅n2

c1⋅n2n2≤3n2n2≤c2⋅n2n2\frac{c1 \cdot n^2}{n^2} \leq \frac{3n^2}{n^2} \leq \frac{c2 \cdot n^2}{n^2} n2c1⋅n2​≤n23n2​≤n2c2⋅n2​

c1≤3≤c2c1 \leq 3 \leq c2 c1≤3≤c2

There exists a constant c1 and c2 such that c1≤3≤c2. Therefore, Θ(3n2)∈Θ(n2)\text{There exists a constant } c1 \text{ and } c2 \text{ such that } c1 \leq 3 \leq c2 \text{. Therefore, } \Theta(3n^2) \in \Theta(n^2) There exists a constant c1 and c2 such that c1≤3≤c2. Therefore, Θ(3n2)∈Θ(n2)

Here, Θ(3n2)\Theta(3n^2)Θ(3n2) has the same growth rate as Θ(n2)\Theta(n^2)Θ(n2), with the only key difference being the constant 333. Therefore, it would be true.

From this, we know that g(x)g(x)g(x) should have an equal growth rate to f(x)f(x)f(x) for it to be a tight bound for f(x)f(x)f(x).

Noting this and our limit definition, we can say that:

If lim⁡x→+∞f(x)g(x)=L, where L>0 and L is a constant (finite), then the function Θ(g(x)) can be considered a tight bound for f(x)\text{If } \lim_{x\to+\infty} \frac{f(x)}{g(x)} = L \text{, where } L > 0 \text{ and } L \text{ is a constant (finite), } \\ \text{then the function } \Theta(g(x)) \text{ can be considered a tight bound for } f(x) If x→+∞lim​g(x)f(x)​=L, where L>0 and L is a constant (finite), then the function Θ(g(x)) can be considered a tight bound for f(x)

Tips

Since Big Θ\ThetaΘ is a tight bound, any function g(x)g(x)g(x) that is considered as a tight bound for f(x)f(x)f(x) is also a possible upper bound (Big OOO) and lower bound (Big Ω\OmegaΩ).

Big Ω notation

Big Ω\OmegaΩ, or Big Omega, represents the best-case time complexity of an algorithm, where f(x)f(x)f(x) is our algorithm, and g(x)g(x)g(x) is our "lower bound". This is basically the opposite of Big O. Here, we wan't g(x)g(x)g(x) to be lower than f(x)f(x)f(x), since g(x)g(x)g(x) is our lower bound.

Formally, to show that f(x)∈Ω(g(x))f(x) \in \Omega(g(x))f(x)∈Ω(g(x)), we prove that:

f(x)≥c⋅g(x)for all x≥x0,where c>0 and x0 is a constantf(x) \geq c \cdot g(x) \hspace{1em} \text{for all}~x \geq x_0, \text{where } c > 0~\text{and}~x_0~\text{is a constant} f(x)≥c⋅g(x)for all x≥x0​,where c>0 and x0​ is a constant

For example, let's try proving that Ω(n2)∈Ω(n)\Omega(n^2) \in \Omega(n)Ω(n2)∈Ω(n):

If f(x)∈Ω(g(x)), then f(x)≥c⋅g(x) for all x≥x0,where c>0 and x0 is a constant\text{If } f(x) \in \Omega(g(x)) \text{, then } f(x) \geq c \cdot g(x) \text{ for all}~x \geq x_0, \text{where } c > 0~\text{and}~x_0~\text{is a constant} If f(x)∈Ω(g(x)), then f(x)≥c⋅g(x) for all x≥x0​,where c>0 and x0​ is a constant

f(x)≥c⋅g(x)f(x) \geq c \cdot g(x) f(x)≥c⋅g(x)

n2≥c⋅nn^2 \geq c \cdot n n2≥c⋅n

n2n≥c⋅nn\frac{n^2}{n} \geq \frac{c \cdot n}{n} nn2​≥nc⋅n​

n≥cn \geq c n≥c

There exists a constant c such that n≥c. Therefore, Ω(n2)∈Ω(n)\text{There exists a constant } c \text{ such that } n \geq c \text{. Therefore, } \Omega(n^2) \in \Omega(n) There exists a constant c such that n≥c. Therefore, Ω(n2)∈Ω(n)

Obviously, a linear function grows lesser than a quadratic one, therefore, Ω(n)\Omega(n)Ω(n) can be a lower bound for Ω(n2)\Omega(n^2)Ω(n2). Thus, Ω(n2)∈Ω(n)\Omega(n^2) \in \Omega(n)Ω(n2)∈Ω(n).

From this, we know that f(x)f(x)f(x) should obviously be greater than or equal to g(x)g(x)g(x) for g(x)g(x)g(x) to be a possible lower bound for f(x)f(x)f(x).

Noting this and our limit definition, we can say that:

If lim⁡x→+∞f(x)g(x)≥1, then the function Ω(g(x)) can be considered a lower bound for f(x)\text{If } \lim_{x\to+\infty} \frac{f(x)}{g(x)} \geq 1 \text{, then the function } \Omega(g(x)) \text{ can be considered a lower bound for } f(x) If x→+∞lim​g(x)f(x)​≥1, then the function Ω(g(x)) can be considered a lower bound for f(x)

Just like before, this limit also fits our requirements, as if it's equal to 111, it would indicate that f(x)f(x)f(x) and g(x)g(x)g(x) grow at the same rate as x→∞x \to \inftyx→∞. If it's greater than 111 or +∞+\infty+∞, we know that the denominator: g(x)g(x)g(x) grows slower than the numerator: f(x)f(x)f(x), thus, allowing it to be considered as a lower bound.

Why ∈?

If you remembered sets from Mathematics, the symbol ∈\in∈ means that an element is part of a set. What set? The set of functions with that "complexity".

For example, let's take our function from before: f(x)=x2+4x+16f(x) = x^2+4x+16f(x)=x2+4x+16. Remember that Big O means the upper bound on the function. Since our function has a growth of x2x^2x2 (largest growing term), it's included in the set of O(n2)O(n^2)O(n2), because n2n^2n2 can be an upper bound on the function f(x)f(x)f(x). Therefore, we can say that x2+4x+16∈O(n2)x^2+4x+16 \in O(n^2)x2+4x+16∈O(n2).

What about O(n)∈O(n2)O(n) ∈ O(n^2)O(n)∈O(n2)? Well, this simply means if the set of functions whose upper bound is O(n)O(n)O(n) can have an upper bound of O(n2)O(n^2)O(n2). The answer? Yes, because O(n2)O(n^2)O(n2) is an upper bound. If a function's upper bound is already O(n)O(n)O(n), then anything greater than or equal to the growth of O(n)O(n)O(n) can be an upper bound, since the only requirement is that it must not exceed it. If a function has an upper bound of O(n)O(n)O(n), it most certainly can have an O(n2)O(n^2)O(n2) one, because it's growth can't exceed O(n)O(n)O(n) already, what more about O(n2)O(n^2)O(n2)?
